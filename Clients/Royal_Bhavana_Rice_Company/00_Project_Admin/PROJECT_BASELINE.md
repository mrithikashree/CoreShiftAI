\# 1. Purpose



This document defines the official baseline for the Royal Bhavana Rice Company Digital Sales \& Lead Management Platform.



It establishes the approved project structure, documentation standards, naming conventions, development workflow, Git strategy, and governance process.



Once approved, this document becomes the single source of truth for the project.



Any future structural changes must be documented through a new baseline version.

\# 9. Naming Standards



Folders



Use Title Case with underscore.



Example:

File Names



Use UPPER\_CASE with underscore.



Example:

Markdown Files



Use .md extension.



Git Repository



Maintain consistent naming across GitHub and local repository.
