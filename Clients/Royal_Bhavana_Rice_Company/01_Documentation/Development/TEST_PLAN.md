\# Royal Bhavana Rice Company

\# Test Plan



\---



\# Purpose



This document defines the testing strategy for the Royal Bhavana Rice Company Digital Sales \& Lead Management Platform.



The objective is to verify that all features work correctly before production deployment.



\---



\# Test Scope



The following modules will be tested:



\- Landing Website

\- Google Form

\- Google Sheets CRM

\- Reports

\- Mobile Responsiveness

\- User Experience



\---



\# Website Testing



\## Header



\- \[ ] Logo displays correctly

\- \[ ] Navigation links work

\- \[ ] WhatsApp button opens chat

\- \[ ] Buy Online button redirects correctly



\---

\- \[ ] Banner loads

\- \[ ] Responsive layout



\---



\## Product Section

\- \[ ] Buttons work



\---



\## Contact Section

\# User Acceptance Testing (UAT)



Client should verify:



\- Branding

\- Content

\- Navigation

\- Products

\- Contact Process

\- Overall Experience



\---



\# Bug Tracking



For every issue record:



\- Bug ID

\- Module

\- Description

\- Severity

\- Status

\- Resolution Date



\---



\# Release Checklist



Before production:



\- \[ ] All critical bugs fixed

\- \[ ] Testing completed

\- \[ ] Client approval received

\- \[ ] GitHub updated

\- \[ ] Backup created



\---



\# Success Criteria



The project is ready for production only when:



\- All critical tests pass.

\- No blocking issues remain.

\- Client signs off on User Acceptance Testing.

\- Deployment checklist is complete.
